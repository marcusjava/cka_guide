Network policies


Kubernetes network policy define o seguinte comportamento:
- Se não houver regras aplicadas ao pod, todo trafego de entrada e saida é permitido

Por padrão se não houver politica aplicada todos os pods outros pods usando IP ou serviço DNS*
Network policy é a ferramenta principal para segurança em rede kubernetes
Permite facilmente restringir trafego de rede em seu cluster e liberar somente o que for permitido 
Metwork Policies necessita de um plugin, mesmo se voçê aplicar um network policy sem o plugin CNI configurado 
estas não surtirão efeito. Ex de plugins: Calico, Weave, Romana
