


Containers podem especificar request que são os recursos que o sistema deverá garantir ao container e um limit que é o maximo de recursos que o sistema irá permitir que o container utilize. 
kubernetes usa QOS quality of service classes na tomada de decisões sobre pod eviction quando os recursos dos nós são excedidos Pods que necessitam estarem ativos constantemente devem requisitar status GARANTIDO.
 
Existem 3 tipos de QoS - Guaranted, Burstable e BestEffort

Para um POD receber um QoS Guaranted:
- Cada container em um pod precisa ter um memory limit e um memory request com valores iguais.
- Cada container em um pod precisa ter um CPU limit e request com iguais valores.


Para um POD receber um Burstable QoS.
- Ao menos um container no Pod possui memoria ou cpu request ou limit.
- Não deve atender aos criterios do Qos Guaranted

Um pod recebe QoS BestEffort quando não possui cpu ou memory limits ou requests e este é um comportamento a ser evitado pois o scheduler não sabe o quanto de recursos 
o processo necessita e nesse caso os Pods são tratados com prioridade mais baixa e são os primeiros a serem terminados quando não houver recursos.
